Dusty Engine

Dusty is an open source game engine.



« »


Dusty is crossplatform, written in C++, and uses only few crossplaform libraries (SDL, OpenGL, OpenCL and Freetype2). It is designed to be generic, simple and effective.


The particularity of this engine is to feature a raytracer as 3D renderer. Raytracing is slower than the classical method (rasterization), but offers a more realistic image by handling natively shadows, reflection, transparency and refraction. The computation of the image is still done by the GPU. An acceleration structure (Bounding Volume Hierarchy) reduces the number of object to be computed and improves performance.


The physic engine of Dusty is inspired by Box2D and Bullet, the collision detection is continuous to prevent tunneling effect. Dusty also has a network engine, server-client based, that makes creating a multi player game easy.


Currently Dusty is oriented to use voxels, which are really well handled by raytracing.



features:


    GLOBAL

  • realtime dynamic raytracer
  • physic engine with continuous collision detection
  • network engine (UDP, server-client based)
  • ingame console with command line interpreter
  • configurable particle systeme
  • voxel engine
  • multhreading
  • SIMD for vector operation

    RENDERER

  • shadow
  • reflection
  • transparency
  • refraction
  • ambiant occlusion
